Claude Code + Obsidian = AI second brain

Setup recap - jak zapięliśmy cross-projekt knowledge base z auto-updateami w tle.

Setup: 26 kwietnia 2026 · Stack open-source · Koszt $0/mies

Stack

  • Obsidian (free, local-first) - vault to folder z plikami .md na dysku
  • Claude Code Desktop - LLM interface
  • AgriciDaniel claude-obsidian - 11 skills + 4 slash commands w <vault>/.claude/ (łączy Karpathy LLM Wiki pattern + Kepano syntax skills)
  • Obsidian Git plugin - auto-commit/push co 5-10 min
  • Private GitHub repo - sync między urządzeniami + backup

Architektura - tree view

~/Documents/Obsidian/
├── .git/                  ← lokalny git → push do prywatnego repo GitHub
└── <vault-name>/
    ├── CLAUDE.md          ← schema (kontrakt z LLM jak utrzymywać wiki)
    ├── log.md             ← append-only chronological log
    ├── hot.md             ← persistent recent context (<500 słów, cache)
    ├── people/            ← notatki o ludziach
    ├── projects/          ← per projekt: status, decyzje, kontakt
    ├── decisions/         ← decision log (YYYY-MM-DD <topic>.md)
    ├── daily/             ← daily notes (Claude generuje)
    ├── learnings/         ← lessons z sesji (Stop hook auto-extract)
    ├── templates/         ← mail-draft, project-setup, decision-log
    ├── email-archive/     ← cross-projekt mail history
    └── .claude/
        ├── skills/        ← AgriciDaniel (Kepano + Karpathy)
        └── commands/      ← /wiki, /save, /canvas, /autoresearch, /lint

Background mode (3 hooki w ~/.claude/settings.json)

HookCo robi
SessionStart Bash skrypt czyta vault/hot.md → Claude od pierwszej sekundy ma recent context bez recap
Stop (po /exit) Bash skrypt dorzuca ## [YYYY-MM-DD HH:MM] session-end | <projekt> do log.md + auto git commit + push
PostCompact Re-load hot.md po kompakcji rozmowy
Bezpieczne dla limitów / banów Anthropic: skrypty bash są deterministyczne, NIE wywołują Claude API. Zero ryzyka token arbitrage / harness pattern który Anthropic banuje (1.45 mln banów H2 2025).

Globalny ~/.claude/CLAUDE.md (auto-loaded w każdej sesji)

Sekcja "Obsidian Vault" mówi LLM:

  • Lokalizacja vault (cross-projekt - jeden niezależnie od cwd)
  • Co zapisywać auto: decisions, people, projects, mail-out, learnings - immediate sync rule
  • Wikilinks zawsze, frontmatter YAML zawsze, polskie znaki zawsze
  • NIGDY nie wpisuj secretów do vault (App Passwords w macOS Keychain, tokens w ~/mcp-keys/)

Setup krok po kroku (jeśli chcesz u siebie)

  1. Install Obsidian + utwórz vault Pobierz z obsidian.md (free), utwórz nowy vault np. ~/Documents/Obsidian/<nazwa>/. Zero konta wymagane (konto tylko dla Obsidian Sync $4/mies).
  2. Subskrypcja Claude Code Pro lub Max ($20/$200/mies). NIE używaj third-party narzędzi typu OpenCode/Cline z Twoim OAuth - automatyczny ban.
  3. Git init vault + private repo na GitHub 
    cd ~/Documents/Obsidian
git init -b main
gh repo create <username>/Obsidian --private --source=. --push
    .gitignore z exclusion: .DS_Store, .obsidian/workspace*.json, .obsidian/cache/, *.key, *.env, secrets/.
  4. Install AgriciDaniel claude-obsidian do vault 
    git clone https://github.com/AgriciDaniel/claude-obsidian /tmp/co
cp -R /tmp/co/skills <vault>/.claude/
cp -R /tmp/co/commands <vault>/.claude/
    Daje 11 skills (zawiera Kepano obsidian-skills) + 4 slash commands (/wiki, /save, /canvas, /autoresearch).
  5. Install community plugins w Obsidian Settings → Community plugins → Browse:
    • Templater - dynamic templates ze scriptami
    • Dataview - SQL-like queries po notatkach
    • Tasks - TODO management cross-files
    • Obsidian Git - auto-commit/push co 5-10 min
  6. Skonfiguruj Obsidian Git Settings → Obsidian Git: backup interval 5 lub 10 min, Auto push ON, Pull on startup OFF (jeśli 1 urządzenie).
  7. Utwórz schema document vault/CLAUDE.md Twój kontrakt z LLM jak utrzymywać wiki. Per Karpathy: definiuje folder structure, konwencje frontmatter, kiedy zapisywać auto, czego NIE robić (np. zero secretów). Wzór w naszym vault.
  8. Utwórz vault/log.md i vault/hot.md log.md = append-only chronological z parseable prefixes ## [YYYY-MM-DD HH:MM] type | topic.
    hot.md = recent context cache (max 500 słów, overwritten każdą sesją).
  9. Setup hooki w ~/.claude/settings.json Trzy bash skrypty (SessionStart, Stop, PostCompact) - czytanie hot.md, auto log entry, git commit/push. 
    "hooks": {
  "SessionStart": [{
    "matcher": "startup|resume",
    "hooks": [{ "type": "command", "command": "~/.claude/scripts/vault-session-start.sh" }]
  }],
  "Stop": [{
    "hooks": [{ "type": "command", "command": "~/.claude/scripts/vault-session-end.sh", "async": true }]
  }],
  "PostCompact": [{
    "hooks": [{ "type": "command", "command": "~/.claude/scripts/vault-session-start.sh" }]
  }]
}
  10. Update ~/.claude/CLAUDE.md z sekcją "Obsidian Vault" Globalny CLAUDE.md auto-loaded w każdej sesji - mówi LLM gdzie jest vault i co tam zapisywać. Bez tego LLM w innym projekcie nie wie że vault istnieje.
  11. Test + restart Claude Code Hooki aktywne dopiero po restart sesji (settings watcher). Otwierasz nową sesję - SessionStart hook czyta hot.md, masz cross-projekt context od pierwszej wiadomości.
  12. Daily use: vault rośnie sam w tle Pracujesz w dowolnym projekcie → LLM zapisuje ważne rzeczy do vault per immediate sync rule → /exit → Stop hook commituje + pushuje. Co tydzień /lint żeby sprawdzić zdrowie vault (per Karpathy health check).

Daily flow (po setupie)

  1. Otwierasz Claude Code w dowolnym projekcie
  2. SessionStart hook wstrzykuje hot cache → LLM zna kontekst
  3. Pracujesz - LLM zapisuje ważne decyzje/maile/learnings do vault automatycznie
  4. /exit → Stop hook commituje + pushuje
  5. Vault rośnie sam w tle, czytasz gdy szukasz

Efekt

  • Cross-projekt memory - sesja w projekcie A widzi wszystko co działo się w projekcie B
  • Zero curation overhead - LLM zapisuje za Ciebie z każdej sesji
  • Knowledge graph - klikasz w person/projekt → pełna historia, mail history, decyzje
  • Search across all - "kiedy ostatnio rozmawialiśmy o X" jednym query
